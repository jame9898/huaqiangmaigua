这瓜保熟吗？

一个还在测试阶段的小工具。用手机麦克风采集敲击声，本地做 FFT 频谱分析，给出一个频率参考值。声音阈值参考自公开论文，并非精确标尺——我们希望和大家一起测、一起记录，把这把"尺"慢慢校准。

这是一个仍在测试阶段的开源小工具。当前频率阈值（133 / 189 Hz）来源于公开论文的简化转写，不能作为唯一参考标准，不同品种、产地、个体的西瓜实际频率范围也存在差异。我们希望大家在切瓜后回传"检测频率 vs 实际生熟"的数据，让这把"声学标尺"被群众一起校准。
01 / 灵感缘起

夏天买瓜，

02 / 科学原理

一个简化的
频率参考，
不是精确判定。

西瓜被敲击后产生振动，主频率与瓜瓤弹性、含水量相关。我们用 FFT 把 2 秒敲击声转成一个频率数字，对照公开论文给出的参考区间——但这仅仅是参考，不是判定结论。

所有分析都在设备本地完成。Web Audio API 采集 44.1kHz 音频 → FFT 8192 点频谱 → 主频提取 → 多次测量取平均。整个过程不上传任何原始音频，仅在用户主动点"上传共创"时，才把 {品种, 重量, 频率, 判定} 上传到云端。

下方阈值区间参考自《农业工程学报》1999 年王书茂等人对"京欣"西瓜的研究，将固有频率与成熟度指数 f²m^(2/3) 做了简化转写——简化必然带来精度损失，不同品种、产地、个体差异都会影响实际表现。

"西瓜成熟度无损检验的冲击振动方法表明，瓜身振动的基频与瓜瓤弹性、含水量呈定量关系。" — 王书茂等, 《农业工程学报》, 1999
> 189 Hz 偏生 瓜瓤致密，含水量高，敲击声偏脆。
133 – 189 Hz 可能熟 组织相对松软，敲击声偏沉。仅供参考。
< 133 Hz 偏过 可能过熟或局部空心，敲击声闷。
03 / 用户旅程

从一次敲击，
到一份记录。

不需要注册、不上传音频、零硬件门槛。打开网页即用，也可以添加到手机主屏当 PWA 使用，离线也能测瓜。

8424 / 麒麟 / 美都 快捷标签，或手动输入。重量用于后续成熟度指数计算。

STEP 02
拍打
五次以上
倒计时 2 秒内用掌心拍打瓜身腹部，系统自动检测瞬态、提取主频、绘制频谱。

STEP 03

给出判定
取多次测量的平均主频，对照参考区间，输出一个建议结果。一致性高自动放行，差异大则建议重测。

STEP 04

回传数据
切开看实际生熟情况，回传"检测频率 vs 实际结果"，帮助大家一起把这把声学尺校准得更准一点。
04 / 实现细节

把"麦克风录到的杂音"，
处理成可以记录的数据。

不是炫技，只是想把数据采集这一步做得稳一些。哪怕参考阈值还不够准，至少每条上传的数据本身是干净的——后续才有可能被群众智慧校准。

DETAIL · 01

三道简单校验
过滤伪信号

① 敲击瞬态检测：对比峰值帧与环境基线，比值 ≥ 2.5 才算真敲击，过滤纯环境底噪。

② 主峰 / 基线比：主峰能量 ÷ 环境基线 ≥ 2.0，拦截共振型底噪。

③ 声学特征校验：主频在 60–400 Hz 且峰值比 ≥ 4.0，过滤敲木头、敲桌面等非西瓜共鸣声。

DETAIL · 02

本地宽松，
共创严校

本地保存沿用宽松策略——自己留着做练习记录没关系。

公共上传走严格校验：多次测量两两差值 ≤ 10 Hz，且每次频谱都通过声学特征校验，否则会弹窗说明原因并建议重测。

目的只是让共创数据池里的每一条都尽量干净。

DETAIL · 03

开源 PWA
+ 边缘后端

前端单文件 index.html，纯原生 JS + Web Audio，没有引入任何前端框架。

后端使用 Cloudflare Pages Functions + Workers KV，按天归档、35 天 TTL，部署成本接近为零。

整个项目 MIT 协议开源，欢迎 fork 自部署或改进。

05 / 一起共创

数据不准没关系，
关键是一起来记。

目前的阈值数据来自一篇 1999 年的论文，样本有限、品种有限、时间也久了。我们不指望它现在就是准的，但相信只要大家持续回传"检测频率 vs 实际生熟"的对照数据，这把尺子会慢慢更可靠。

COMMUNITY · 西瓜研究员

欢迎你成为
"西瓜研究员"。

不需注册、不收集任何个人信息。上传后系统按当天顺序自动分配编号——001 号、002 号……我们把每一位参与者都叫做"西瓜研究员"。每一条数据都是公共的、匿名的、可被复核的。

真正能让这个项目慢慢走下去的，不是某一个阈值，而是大家愿意把切开后的实际情况回传上来。哪怕只是"这颗 175 Hz 切开是熟的"或者"这颗 145 Hz 实际是生的"，对项目都有价值。
