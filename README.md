# 🍉 敲瓜辨生熟

> 掏出手机敲两下，用频率数据辅助判断西瓜生熟。**共创项目**：欢迎大家上传真实数据共同完善。

一个西瓜声学检测小工具。用手机麦克风采集敲击声，本地 FFT 频谱分析算出主频率，结合多次测量取平均判断生熟。**音频在本地分析，仅当主动点"☁️ 上传共创"时才把 {品种, 频率, 判定} 上传到云端汇总**。可"添加到主屏幕"当 PWA 使用，离线也能测瓜。

🔗 **在线地址**：<https://huaqiangmaigua.pages.dev>

## 📖 频率原理

西瓜被敲击后会产生振动，振动的**主频率（基频）**和成熟度相关：西瓜成熟后瓜瓤弹性模量变小，对质量一定的西瓜而言固有频率降低，因此通过频率可以估算成熟度。

### 🎯 生熟判断阈值（简化参考）

| 平均主频率 | 参考判断 | 含义 |
|-----------|---------|------|
| **> 189 Hz** | 🟢 生瓜（偏生） | 瓜瓤致密、含水量高，振动频率高，"梆梆"响 |
| **133 – 189 Hz** | 🟡 熟瓜（适中） | 瓜瓤糖分转化、组织软化，振动频率适中，"嘭嘭"响 |
| **< 133 Hz** | 🔴 可能过熟/空心 | 瓜瓤过软或空心，振动频率低，"噗噗"闷响 |

### 📚 数据来源与简化说明

本工具的判断思路参考自**《农业工程学报》**公开发表的研究：

> 王书茂, 焦群英, 籍俊杰. 西瓜成熟度无损检验的冲击振动方法[J]. 农业工程学报, 1999, 15(3): 240-245.
> （作者单位：中国农业大学、河北省农业机械化研究所）
> 原文链接：http://www.tcsae.org/cn/article/id/19990350

需要说明的是：
- **原研究使用的是与西瓜质量相关的成熟度指数 f²m^(2/3)**（频率与质量综合判定），并非单一固定频率阈值。
- 本工具为简化易用，**将其简化为固定频率阈值**（133/189Hz），未引入质量修正，因此精度有限。
- 该论文发表于 1999 年，研究品种为"京欣"西瓜，不同品种、产地、个体的实际频率范围可能存在差异。
- **若你需要更准确的数据，欢迎有相关研究或实测数据的朋友贡献补充**，共同完善参考标准。
- **若使用者在实测中发现数据与判断结果存在明显差异，欢迎联系我并上报对应数据**（品种、质量、实测频率、实际生熟情况），以便持续校准阈值。

## 🚀 使用方法

### 手机用户（推荐）
1. 用 **Chrome 或 Edge 浏览器**（别用微信内置浏览器）访问在线网址：
   ```
   https://huaqiangmaigua.pages.dev
   ```
2. **填写品种**：从快捷标签点选（8424 / 麒麟 / 甘美 / 特小凤 / 黑美人 / 京欣 / 美都），或**手动输入自定义品种名**（如：早春红玉、甜王等）
3. **（建议）填写瓜重（kg）**：重量有科学依据——成熟度指数 f²m^(2/3) 与质量相关，建议如实填写（不知可留空）
4. 点"开始测量"→ 允许麦克风 → 倒计时内用**指关节轻敲瓜身中部（腹部）**→ 连续 3 次
5. 看平均频率和判断结果：
   - 点 **💾 保存记录** 仅保存到本机（localStorage）
   - 点 **☁️ 上传共创** 把本次 {品种, 瓜重, 频率, 生熟判定} 上传到云端，匿名加入今日"西瓜研究员"列表

### 共创数据说明
- 上传后系统按当天上传顺序自动分配编号：001 号、002 号……称为"**西瓜研究员**"
- **每条记录包含**：品种、瓜重（如有）、频率、生熟判定、成熟度指数 f²m^(2/3)
- 数据按天归档，每天北京时间 0 点重置编号
- 大家可在"**共创数据**"卡片查看今日上传记录，并切到"**📊 数据分析**"看品种 / 判定结果 / 频率 / 瓜重 / 成熟度指数的实时统计
- **不需要注册、不收集任何个人信息**，请大家上传真实数据共同完善这个项目
- ⚠️ **数据严谨性说明**：瓜重（kg）有科学参考价值（f²m^(2/3) 与质量相关），建议如实填写；品种请填准确名称，不确定可写"未知"；建议同一西瓜测 3 次以上、两两差值 ≤ 10Hz 再上传

### 添加到主屏幕（变成桌面 App）
以 **Edge** 为例（Chrome 操作类似）：
1. 先给浏览器开权限：手机**设置 → 应用管理 → Edge → 权限管理**
   - **麦克风** → 允许（用于采集敲击声）
   - **其他权限 → 桌面快捷方式** → 允许（用于在桌面创建图标）
2. 用 Edge 打开网址，点**右下角三个横杠**（菜单/设置）→ 第一页下方找到 **"添加至手机"** → 确认安装
3. 桌面会出现 **"敲瓜"** 图标（桌面空间有限，显示的是应用简称"敲瓜"），点图标**全屏打开**（无浏览器地址栏），离线也能用

> 验证安装是否成功：点桌面"敲瓜"图标，若**全屏打开、无地址栏**即 PWA 安装成功；若仍带地址栏则只是网页书签，重新走一遍上述步骤。

## 🏗️ 项目架构（自部署 / 二次开发可参考）

```
前端  index.html（单文件 PWA · 纯 HTML + 原生 JS）
        │  Web Audio FFT 本地分析
        │  仅在点击上传时调用 ↓
        ▼
后端  functions/api/*.js（Cloudflare Pages Functions）
        ├─ POST /api/upload   上传一条记录
        ├─ GET  /api/records  当日上传列表（滚动展示）
        └─ GET  /api/analysis 当日聚合统计
                │
                ▼
存储  Cloudflare Workers KV
        ├─ counter:YYYY-MM-DD     当日编号自增器
        └─ uploads:YYYY-MM-DD     当日记录数组（含 weight、ripenessIndex，35 天 TTL）
```

部署方式：fork 本仓库 → 在 Cloudflare Pages 连接你的 fork → Framework preset 选 None、build command 留空 → Settings → Bindings 添加变量名 `KV` 绑定到一个你新建的 KV namespace。详细步骤可在 Issues 区询问。
