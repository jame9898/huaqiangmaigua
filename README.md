# 🍉 敲瓜辨生熟 · 华强买瓜神器

> **"这瓜保熟吗？"**
> —— 现在不用瞪眼看了，掏出手机敲两下，让科学替华强回答这个灵魂拷问。

一个纯前端的西瓜声学检测工具。用手机麦克风采集敲击声，本地 FFT 频谱分析算出主频率，结合多次测量取平均判断生熟。**音频全程本地处理，不上传任何服务器**，双击即用，部署后手机添加到主屏幕就像原生 App，户外买瓜离线也能用。

## 📖 这瓜到底保不保熟？—— 频率原理

西瓜被敲击后会产生振动，振动的**主频率（基频）**和成熟度直接相关。这和拍西瓜听声音是同一个道理，只不过我们把"耳朵听"换成了"麦克风+FFT 量化"。

### 🎯 生熟判断阈值（通用参考）

| 平均主频率 | 参考判断 | 含义 |
|-----------|---------|------|
| **> 189 Hz** | 🟢 生瓜（偏生） | 瓜瓤致密、含水量高，振动频率高，"梆梆"响 |
| **133 – 189 Hz** | 🟡 熟瓜（适中） | 瓜瓤糖分转化、组织软化，振动频率适中，"嘭嘭"响 |
| **< 133 Hz** | 🔴 可能过熟/空心 | 瓜瓤过软或空心，振动频率低，"噗噗"闷响 |

> ⚠️ 以上为**通用参考阈值**，不同品种（8424、麒麟、甘美、特小凤、黑美人等）有差异。本工具给出的是参考性判断，不构成品质保证——毕竟华强也不是每次都看得准。

### 🎵 为什么会测到 447Hz？（踩坑记录）

开发时实测麒麟瓜，三次稳定测得 **447Hz**，按阈值判"生瓜"。但切开一看明明是熟瓜——问题出在哪？

物理上一个物体被敲击会同时发出多个频率：**基频**（我们想要的）+ 一系列**谐波**（2倍、3倍…）。447Hz 恰好是 **149Hz 的 3 次谐波**（447 ÷ 3 ≈ 149Hz，正是熟瓜范围）。初版算法在 80–500Hz 全段取能量最大值，被能量更强的 3 次谐波"骗"了。

**修复方案**：改为"低频基频优先"——先在 60–250Hz 基频区找峰值，只有低频确实无信号才回退到 100–500Hz。修复后实测同瓜降到 176Hz，正确判定为熟瓜。**这段踩坑也证明了：稳定重复的异常值，往往不是随机误差，而是算法解读方式的问题。**

## 🧭 Roadmap 设计思路

本项目源自一份可执行的最小可行产品（MVP）路线图，核心思路如下：

### 核心功能清单
- 🎤 **录音采样**：调用麦克风录制敲击声，Web Audio API 本地 FFT 频谱分析，计算主频率（Hz）。音频不上传。
- 🔢 **多次测量**：同一瓜至少测 3 次（默认 3 次，可增加到 8 次），每次独立录音分析，取平均降低误差。
- 📊 **一致性判定**：各次测量两两差值 ≤ 10Hz 时结果确定；差异明显（>10Hz）则提示重新测量，并标出偏离最大的那次。
- 🏷️ **品种记录**：文本输入 + 常见品种快捷选择（8424/麒麟/甘美/特小凤/黑美人）。
- 📋 **结果展示**：展示平均频率、各次原始频率、生熟判断徽章、通用阈值说明。
- 💾 **数据暂存**：localStorage 本地保存历史记录（品种、各次频率、平均值、判断、时间戳），支持单条删除与清空。

### 技术选型与限制
- **纯前端**：HTML + CSS + JavaScript，单文件交付，无后端依赖、无构建步骤。
- **核心 API**：Web Audio API（`AudioContext`、`AnalyserNode`、`getUserMedia`）。
- **数据存储**：浏览器 `localStorage`。
- **浏览器要求**：现代 Chrome / Edge / Firefox / Safari。麦克风需 HTTPS 或 localhost 环境授权。

### 从路线图到上线的演进
1. 搭建 HTML 骨架 → 实现录音与 FFT 分析 → 多次测量循环 → 平均值与偏差校验 → 保存与历史展示
2. 低频基频优先算法修复（447Hz 谐波踩坑）
3. 麦克风会话级复用（避免每次敲击重新授权）
4. 测量次数下限提到 3，10Hz 一致性判定
5. 移动端适配 + 耳机麦克风优先
6. PWA 化（manifest + service worker 离线缓存）+ 双部署（GitHub Pages / Gitee Pages）

## 🔧 技术实现要点

### 音频分析管线
```
麦克风 → 高通滤波(50Hz) → 低通滤波(500Hz) → AnalyserNode(FFT 8192) → 累积能量谱 → 低频优先峰值 → 抛物线插值 → 主频率
```

- **FFT 大小 8192**：频率分辨率 ≈ 5.4Hz（44100/8192），能精细区分 133Hz 与 189Hz 临界值。
- **带通滤波 50–500Hz**：滤除低频环境隆隆声与高频人声/摩擦噪，专注西瓜共振频段。
- **累积能量谱**：录音窗口内逐帧累加各频段能量，抑制随机噪声、突出共振峰值。
- **低频基频优先**：先在 60–250Hz 找峰值，只有低频无有效信号才回退到 100–500Hz，避免误抓高次谐波。
- **抛物线插值**：在峰值 bin 的左右邻居间做二次曲线拟合，把频率精度从整数 bin 提升到子 bin 级别。

### 一致性判定逻辑
- 计算所有**两两差值的最大值** maxDiff（等价于排序后最大值-最小值，即极差）。
- maxDiff ≤ 10Hz → 一致，结果确定（绿色 ✅）。
- maxDiff > 10Hz → 差异明显，标记偏离中位数最大的那次为异常（橙色 ⚠️），建议重新测量。
- **10Hz 的由来**：按 176Hz 熟瓜计，10Hz 两两差值 ≈ 5.7% 相对偏差，比 ±15% 严、比 5Hz 宽，卡在"能容忍正常敲击波动，又不会放过明显异常"的甜区。

### 耳机麦克风优先
- `enumerateDevices` 检测耳机/蓝牙/USB/通讯设备，优先用其 `deviceId`。
- 适合把手机放桌上、耳机线凑近瓜的户外场景（后期隐蔽使用基础）。

### PWA 离线
- `manifest.webmanifest`：应用名、西瓜图标、全屏 standalone。
- `sw.js`：Service Worker 缓存核心文件，添加到主屏幕后离线可用。
- `icon.svg`：矢量西瓜图标，任意尺寸清晰。

## ⚠️ 注意事项

- **麦克风需安全上下文**：HTTPS 或 localhost。`file://` 直接打开多数浏览器禁用麦克风，需部署或起本地服务器。
- **务必用 Chrome**：国产浏览器（AG 等）对 Web Audio API 和 PWA 支持不完整，Chrome / Edge / Safari 体验最佳。
- **别用微信内置浏览器**：会屏蔽 github.io 等域名，显示 404。
- **敲击技巧**：用**指关节**（不是指甲）轻敲瓜身**中部（腹部）**，力度适中，周围尽量安静。人声（100–300Hz）会干扰测量。
- **通用参考**：阈值是通用的，不同品种有差异，仅供参考。本工具不构成品质保证。
