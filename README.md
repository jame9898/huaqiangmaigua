# 🍉 这瓜保熟吗

> 掏出手机拍两下，用频率数据辅助判断西瓜生熟。**共创项目**：欢迎大家上传真实数据共同完善。

一个西瓜声学检测小工具。用手机麦克风采集拍打声，本地 FFT 频谱分析算出主频率，结合多次测量取平均判断生熟。**音频在本地分析，仅当主动点"☁️ 上传共创"时才把 {品种, 重量, 频率, 判定} 上传到云端汇总**。可"添加到主屏幕"当 PWA 使用，离线也能测瓜。

## 📖 频率原理

西瓜被拍打后会产生振动，振动的**主频率（基频）**和成熟度相关：西瓜成熟后瓜瓤弹性模量变小，对质量一定的西瓜而言固有频率降低，因此通过频率可以估算成熟度。

### 🎯 生熟判断阈值（简化参考）

| 平均主频率 | 参考判断 | 含义 |
|-----------|---------|------|
| **> 189 Hz** | 🟢 生瓜（偏生） | 瓜瓤致密、含水量高，振动频率高，"梆梆"响 |
| **133 – 189 Hz** | 🟡 熟瓜（适中） | 瓜瓤糖分转化、组织软化，振动频率适中，"嘭嘭"响 |
| **< 133 Hz** | 🔴 可能过熟/空心 | 瓜瓤过软或空心，振动频率低，"噗噗"闷响 |

> **一致性建议**：同一西瓜建议测量 5 次以上，各次测量两两差值 ≤ 10Hz 时结果较可靠；若差异明显（>10Hz），建议重新测量。

### 📚 数据来源与简化说明

本工具的判断思路参考自**《农业工程学报》**公开发表的研究：

> 王书茂, 焦群英, 籍俊杰. 西瓜成熟度无损检验的冲击振动方法[J]. 农业工程学报, 1999, 15(3): 240-245.
> （作者单位：中国农业大学、河北省农业机械化研究所）
> 原文链接：http://www.tcsae.org/cn/article/id/19990350

需要说明的是：
- **原研究使用的是与西瓜质量相关的成熟度指数 f²m^(2/3)**（频率与质量综合判定），并非单一固定频率阈值。
- 本工具为简化易用，**将其简化为固定频率阈值**（133/189Hz），未引入质量修正，因此精度有限。
- 该论文发表于 1999 年，研究品种为"京欣"西瓜，不同品种、产地、个体的实际频率范围可能存在差异。
- **若你需要更准确的数据，欢迎有相关研究或实测数据的朋友贡献补充**，共同完善参考标准。
- **若使用者在实测中发现数据与判断结果存在明显差异，欢迎联系我并上报对应数据**（品种、质量、实测频率、实际生熟情况），以便持续校准阈值。

## 🚀 使用方法

### 手机用户（推荐）
1. 用 **Chrome 或 Edge 浏览器**（别用微信内置浏览器）访问在线网址：
   ```
   https://huaqiangmaigua.pages.dev
   ```
2. **填写品种**：从快捷标签点选（8424 / 麒麟 / 美都 / 未知品种），或**手动输入**
3. **填写西瓜重量（kg）**：保存可留空，上传时必填（0–10 kg）
4. 点"开始测量"→ 允许麦克风 → 倒计时内用**掌心拍打瓜身腹部**→ 连续 5 次
5. 每次测量完成后会**实时绘制对应数量的频谱图**（0–500Hz 范围累积能量谱，主峰红色高亮，标注峰值频率与峰值比），测量几次生成几张
6. 看平均频率和判断结果：
   - 点 **💾 保存记录** → 弹窗确认（可选填"切开结果"）→ 存到本机（**无校验限制**）
   - 点 **☁️ 上传共创** → 通过质量校验后弹窗确认 → 选择"切开后的实际结果"（生瓜 / 熟瓜 / 过熟/空心）→ 上传到云端，同时存到本机

### 🔒 公共上传数据校验

为保证共创数据质量，**公共上传**会进行严格校验，**本地保存**则不受限制。

| 校验项 | 规则 | 不达标处理 |
|--------|------|-----------|
| 测量次数 | ≥ 5 次 | 弹窗提示，不可上传 |
| 一致性 | 多次测量两两差值 ≤ 10Hz | 弹窗提示，不可上传 |
| 频谱特征 | 主频 60–250Hz 共振区，主峰突出度 ≥ 10（**参考提示，不拦截上传**） | 仅作参考提示 |

> **说明**：声学特征校验已降级为参考提示（BETA 阶段），不再作为上传拦截条件。频谱异常时仍可在测量结果中看到「⚠ 声学特征仅供参考」标签，但不影响本地保存与公共上传。

校验不达标（测量次数或一致性）时会**弹出红色提示弹窗**，列出具体原因（差值多少 Hz），并提供两个选项：
- **「仍要保存到本地」**：本地保存不受限，可继续记录
- **「重新测量」**：直接触发新一轮测量，按规范重新拍打

> **声学特征判定原理**：西瓜拍打声应有明显的共振主峰，频谱能量集中在主频附近。主峰突出度 = 主峰单 bin 能量 / 全频段各 bin 平均能量，反映主峰的尖锐程度。实测有效拍打突出度通常在 30–80 之间，阈值暂定 ≥ 10（后续按实测数据调整）。主频搜索范围为 60–250Hz 共振区（fallback 100–500Hz），若主峰不突出或主频超出合理共振范围，会标注「声学特征仅供参考」。

### 🎯 拍打检测原理

为避免把环境底噪（空调、风扇、电源谐波等）误判为拍打声，每次测量会进行**两道检测**：

1. **拍打瞬态检测**：计算 2 秒录音中每帧能量，取峰值帧与最低 30% 帧（环境基线）的比值。纯环境底噪各帧能量接近，比值≈1；真实拍打有明显的能量突增，比值 ≥ 2.5。比值不足则判定为"未检测到拍打声"。
2. **频谱声学特征**：瞬态通过后，再校验主频范围和主峰突出度（见上文）。

**测量 tips**：
- 在相对安静的环境下测量，避免空调/风扇直吹麦克风
- 拍打时用**掌心**拍打西瓜腹部，力度适中
- 持续说话、音乐等非瞬态声音会被瞬态检测拦截
- 拍打桌子、沙发等非西瓜物体可能通过瞬态检测，但会被频谱特征校验拦截（主峰突出度不足）

## 🍉 综合判断建议（重要）

**频率仅反映西瓜成熟度，不代表西瓜品质。** 声学检测只能回答"熟不熟"，回答不了"好不好吃"。购买前请同时观察以下维度：

| 维度 | 观察要点 |
|------|---------|
| **外表** | 是否有病斑、机械伤、日烧痕；花纹是否清晰饱满 |
| **瓜蒂** | 新鲜度（绿色/干枯）、弯曲度（弯曲通常表示自然成熟） |
| **瓜尾** | 切开前观察瓜尾（脐部对面）是否有异常泛白——可能是葫芦素渗出 |
| **天气** | 产地近期是否有降雨、气温是否异常（高温/低温） |
| **种植方式** | 露天种植 vs 大棚种植——露天种植受天气影响更大 |
| **销售半径** | 运输距离远近——长途运输的瓜可能经过冷藏/催熟处理 |

### ⚠️ 葫芦素风险提示

**露天种植且遭遇降雨（尤其是连续降雨或雨灾）的西瓜，可能产生葫芦素。** 葫芦素是西瓜在逆境下的应激产物，会带来苦味甚至引起肠胃不适，**声学检测无法识别**。

判断建议：
- 优先选择**大棚种植**或**近期无降雨地区**的西瓜
- 询问商家**进货时间**——积压库存风险更高
- 切开后若瓜尾泛白、有苦味，立即停止食用
- 综合考虑**销售半径**与**产地天气**——本地应季 + 短途运输通常风险最低

> 频率数据 + 外表观察 + 天气信息 = 更可靠的购买决策。声学只是其中一环。

## 🏗️ 项目架构（自部署 / 二次开发可参考）

```
前端  index.html（单文件 PWA · 纯 HTML + 原生 JS）
        │  Web Audio FFT 本地分析
        │  测量后实时绘制频谱图（Canvas，按测量次数生成）
        │  公共上传前进行质量校验（一致性 + 声学特征）
        │  仅在点击上传且校验通过时调用 ↓
        ▼
后端  functions/api/*.js（Cloudflare Pages Functions）
        ├─ POST /api/upload   上传一条记录
        ├─ GET  /api/records  当日上传列表（滚动展示）
        └─ GET  /api/analysis 当日聚合统计
                │
                ▼
存储  Cloudflare Workers KV
        ├─ counter:YYYY-MM-DD     当日编号自增器
        └─ uploads:YYYY-MM-DD     当日记录数组（含 weight、ripenessIndex，35 天 TTL）
```

### 本地开发
```bash
# 静态预览（仅前端，麦克风可用，上传 API 不可用）
python -m http.server 8000
# 访问 http://localhost:8000

# 完整带后端（需安装 wrangler）
npx wrangler pages dev . --kv KV
```

部署方式：fork 本仓库 → 在 Cloudflare Pages 连接你的 fork → Framework preset 选 None、build command 留空 → Settings → Bindings 添加变量名 `KV` 绑定到一个你新建的 KV namespace。详细步骤可在 Issues 区询问。
